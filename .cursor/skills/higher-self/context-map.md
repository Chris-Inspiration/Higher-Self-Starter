# Context Map — Where Everything Lives

This is the single reference for locating any type of information. When you need context, consult this map instead of guessing or re-reading raw files.

## .cursor/ — Skills & Rules (The Operating System)

| Path | Contents | When to Read |
|------|----------|-------------|
| `.cursor/rules/higher-self.mdc` | Always-active rule: read the Higher Self skill every conversation | Automatic |
| `.cursor/rules/privacy-boundary.mdc` | Always-active rule: never expose private content in public files | Automatic |
| `.cursor/rules/self-verification.mdc` | Always-active rule: verify self-consistency | Automatic |
| `.cursor/skills/higher-self/` | The Higher Self skill: SKILL.md, protocols.md, context-map.md (this file) | Every conversation |

## HIGHER SELF/self/ — Private Operational Layer

| Path | Contents | When to Read |
|------|----------|-------------|
| `HIGHER SELF/self/state.md` | Current state: what's happening now, active focus, open questions, floating tasks | **Every conversation** |
| `HIGHER SELF/self/projects/` | One file per project — status, next steps, blockers | When discussing projects or deciding what to work on |
| `HIGHER SELF/self/patterns/` | Named behavioral patterns — what they look like, what's underneath, antidotes | When you notice a pattern repeating |
| `HIGHER SELF/self/people/` | People in your life worth remembering context on | When someone is mentioned that should be remembered |
| `HIGHER SELF/self/life/` | Location, logistics, daily structure, practical situation | When advising on daily structure or energy |
| `HIGHER SELF/self/activities/` | Activity tracker — what was done and when | When checking consistency or recent history |
| `HIGHER SELF/self/processed/` | Summary files for each piece of intake. Named `processed-NNN_YYYY-MM-DD_type_title-slug.md` — see `CONVENTION.md` in that folder | When you need context on what has been consumed/processed |
| `HIGHER SELF/self/research/` | Ongoing investigations — topics being explored in depth | When doing research or connecting dots |
| `HIGHER SELF/self/medicine/` | Collected truths and wisdom — things that shift state when read at the right moment | When a pattern is active or processing difficult material |

## HIGHER SELF/RAW Input/ — Unprocessed Source Material

Everything raw lives here. Space is cheap, context is not. Always use `HIGHER SELF/self/processed/` summaries rather than re-reading raw files.

| Path | Contents |
|------|----------|
| `HIGHER SELF/RAW Input/Intake/` | Dumped transcripts, articles, anything being consumed. After processing, move to `Intake/processed/` |
| `HIGHER SELF/RAW Input/Videos/` | Voice memos, video journals — transcripts of you talking things through |
| `HIGHER SELF/RAW Input/Cursor/` | Cursor conversation exports |

## Key Principle: DRY (Don't Repeat Yourself)

Never duplicate information. Each type of information lives in ONE place:

| Type | Source of Truth |
|------|----------------|
| Current state & focus | `HIGHER SELF/self/state.md` |
| Patterns | `HIGHER SELF/self/patterns/` |
| Activities | `HIGHER SELF/self/activities/` |
| Floating tasks | state.md "Floating Tasks" section |
| Projects | `HIGHER SELF/self/projects/` |
| Processed intake | `HIGHER SELF/self/processed/` |
